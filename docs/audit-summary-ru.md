# Итоги быстрого аудита фронтенда

- В коде использовались импорты `@/lib/api`, но самого модуля не было. Из-за этого сборка фронтенда ломалась на этапе резолва модулей. Добавлен единый хелпер `src/lib/api.ts`, который нормализует базовый URL из `NEXT_PUBLIC_API_URL`, хранит API‑ключ в `localStorage`, выставляет заголовок `X-API-Key` и предоставляет обёртку `apiFetch` для всех запросов. Теперь страницы (напр. `/auth` и `settings`) получают ключ из одного места и автоматически подмешивают его в запросы.
- Страница `/auth` остаётся точкой входа для проверки и сохранения ключа. Она использует новые функции `setApiKey`, `clearApiKey` и `getAuthHeaders`, чтобы валидировать ключ через эндпоинт `/api/settings/system` и держать активный ключ в браузере.

Если нужны дополнительные правки или перевод остальных экранов, дайте знать.
